## 一、项目概述

AI 创作工坊是一个基于百炼 API 的 AI 创作 SaaS 平台，支持文生图、文生视频、图生视频等功能。用户通过积分消费模式使用各类 AI 模型。

### 核心功能
- ✅ 用户注册/登录（JWT 认证）
- ✅ 文生图（WanX 2.1，消耗 10 积分）
- ✅ 文生视频（WanX 2.1，消耗 50 积分/5秒）
- ✅ 图生视频（上传图片生成视频，消耗 50 积分/5秒）
- ✅ 作品库管理（收藏、标签、删除）
- ✅ 历史记录查询
- ✅ 使用统计概览
- ✅ 管理员后台（用户管理、任务管理、充值、统计分析、模型配置）

## 四、快速启动

### 本地开发

```bash
# 1. 安装依赖
cd backend && npm install
cd ../frontend && npm install

# 2. 配置环境变量
# 编辑 backend/.env
# BAILIAN_API_KEY=sk-xxxxxxxx
# JWT_SECRET=your_secret
# PORT=3001

# 3. 启动后端
cd backend
node server.js

# 4. 启动前端（新终端）
cd frontend
npm run dev
# 访问 http://localhost:5173
```

### 生产部署

```bash
# 构建前端
cd frontend
npm run build
# 输出到 frontend/dist/

# 上传到服务器
scp -r frontend/dist/* root@101.200.84.23:/www/wwwroot/ai-media-studio/

# 重启 PM2
pm2 restart ai-media-studio
```

## 五、API 接口清单

### 认证接口
| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| POST | `/api/auth/register` | 注册 | `{ email, password, nickname? }` |
| POST | `/api/auth/login` | 登录 | `{ email, password }` |
| GET | `/api/auth/me` | 获取当前用户 | Header: `Authorization: Bearer <token>` |

### 生成接口（需认证）
| 方法 | 路径 | 说明 | 参数 |
|------|------|------|------|
| POST | `/api/generate/text-to-image` | 文生图 | `{ prompt, planKey, aspectRatio, n }` |
| POST | `/api/generate/text-to-video` | 文生视频 | `{ prompt, planKey, duration, ratio }` |
| POST | `/api/generate/image-to-video` | 图生视频 | `FormData: image, prompt, planKey, duration` |

### 任务与作品
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/task/:taskId` | 查询任务状态 |
| GET | `/api/tasks` | 用户任务历史 |
| GET | `/api/gallery` | 用户作品库 |
| PUT | `/api/gallery/:id` | 更新作品（标题/标签） |
| DELETE | `/api/gallery/:id` | 删除作品 |

### 管理后台接口（需管理密码）
在请求头中添加 `X-Admin-Password: admin123`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/admin/stats` | 全局统计 |
| GET | `/api/admin/users` | 用户列表 |
| GET | `/api/admin/tasks` | 任务列表 |
| GET | `/api/admin/gallery` | 作品列表 |
| POST | `/api/admin/config/providers` | 添加/更新服务商 |
| POST | `/api/admin/config/models` | 添加/更新模型 |
| DELETE | `/api/admin/config/providers/:id` | 删除服务商 |
| DELETE | `/api/admin/config/models/:id` | 删除模型 |

## 六、配置文件说明

### `backend/config.json`
**重要**：此文件控制所有 AI 模型和服务商配置，修改后无需重启服务。

```json
{
  "providers": [
    {
      "id": "bailian",
      "name": "阿里云百炼",
      "baseUrl": "https://dashscope.aliyuncs.com",
      "apiKey": "sk-xxxxxxxx",
      "enabled": true
    }
  ],
  "models": [
    {
      "id": "wan2.7-image",
      "providerId": "bailian",

      "label": "WanX 2.7 文生图",
      "price": 10,
      "enabled": true

  ]
}
```

## 七、数据库 schema

### `users` 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | UUID |
| email | TEXT | 邮箱（唯一） |
| password | TEXT | bcrypt 哈希 |
| nickname | TEXT | 昵称 |
| balance | REAL | 余额（元） |
| createdAt | TEXT | 注册时间 |

### `tasks` 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | UUID |
| userId | TEXT | 用户 ID |
| type | TEXT | text-to-image / text-to-video / image-to-video |
| planKey | TEXT | 模型 ID |
| status | TEXT | pending / running / succeeded / failed |
| prompt | TEXT | 提示词 |
| result | TEXT | JSON 数组（结果 URL） |
| costYuan | REAL | 消耗金额 |
| createdAt | TEXT | 创建时间 |

### `gallery` 表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | UUID |
| userId | TEXT | 用户 ID |
| type | TEXT | 作品类型 |
| result | TEXT | JSON 数组（结果 URL） |
| tags | TEXT | JSON 数组（标签） |
| createdAt | TEXT | 创建时间 |
